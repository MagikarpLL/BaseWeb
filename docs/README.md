# 个人网站项目文档

> 本目录按 **Harness 工程思想** 拆分为三个层次 + 元规范：
> - **Requirement**：回答"做什么" — 用户故事、页面布局、交互行为、验收标准
> - **Spec**：回答"怎么做" — API 契约、数据库模型、代码规范、技术架构
> - **Plan**：回答"什么时候做" — 迭代计划、任务拆分、验收追溯矩阵
> - **元规范**：文档怎么写、术语怎么用、模板怎么套

## 拆分原则

|| 原则 | 说明 |
||------|------|
|| Requirement 只说"做什么" | 功能概述、页面布局、交互行为、验收标准 |
|| Spec 只说"怎么做" | API 端点、JSON 结构、数据库 DDL、代码实现 |
|| Plan 只说"何时做" | 迭代划分、任务拆分、优先级、依赖关系 |
|| 不重复 | 同一信息只在一个地方维护 |
|| 双向可追溯 | 每条 Spec 可追溯到对应的 Requirement，每条任务对应到 Requirement 和 Spec |
|| 遵循元规范 | 所有文档遵循 CONVENTIONS.md 编写规范，使用 GLOSSARY.md 统一术语 |
