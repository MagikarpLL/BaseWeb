# 04-认证与安全

## 4.1 JWT Token 设计

### Access Token

| 字段 | 值 |
|------|------|
| 有效期 | 15 分钟 |
| 存储 | 前端内存（不持久化） |
| Header | `Authorization: Bearer <token>` |

### Refresh Token

| 字段 | 值 |
|------|------|
| 有效期 | 7 天 |
| 存储 | HTTP-only Cookie |
| 传输方式 | 自动随请求发送（credentials: include） |
| 刷新接口 | POST /api/auth/refresh |

## 4.3 安全措施

| 措施 | 说明 |
|------|------|
| 密码加密 | BCrypt |
| HTTPS | 生产环境必须使用 |
| HTTP-only Cookie | 防止 XSS 攻击获取 Refresh Token |
| 短期 Access Token | 减少 Token 泄露风险 |
| 请求限流 | 防止暴力破解 |
| SQL 注入防护 | MyBatis 参数化查询 `#{}`，禁止 `${}` |
| XSS 防护 | 前端转义 + CSP |
| CSRF 防护 | SameSite Cookie 属性 |

## 4.4 前端会话管理实现

### Pinia 认证 Store

```typescript
// stores/auth.ts
export const useAuthStore = defineStore('auth', {
  state: () => ({
    accessToken: null as string | null,
    user: null as { id: number; username: string; role: string } | null,
    isAuthenticated: false,
  }),

  getters: {
    isAdmin: (state) => state.user?.role === 'ADMIN',
  },

  actions: {
    setAuth(accessToken: string, user: any) {
      this.accessToken = accessToken
      this.user = user
      this.isAuthenticated = true
    },
    clearAuth() {
      this.accessToken = null
      this.user = null
      this.isAuthenticated = false
    },
  },
})
```

### Axios 拦截器配置

```typescript
// utils/api.ts
const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  withCredentials: true,  // 发送 Refresh Token Cookie
})

// 请求拦截器：附加 Access Token
api.interceptors.request.use((config) => {
  const authStore = useAuthStore()
  if (authStore.accessToken) {
    config.headers.Authorization = `Bearer ${authStore.accessToken}`
  }
  return config
})

// 响应拦截器：处理 401 刷新 Token
api.interceptors.response.use(
  (response) => response,
  async (error) => {
    const originalRequest = error.config

    if (error.response?.status === 401 && !originalRequest._retry) {
      originalRequest._retry = true

      try {
        const res = await axios.post('/api/auth/refresh', {}, {
          withCredentials: true,
        })
        const { access_token } = res.data.data

        const authStore = useAuthStore()
        authStore.setAuth(access_token, authStore.user)

        originalRequest.headers.Authorization = `Bearer ${access_token}`
        return api(originalRequest)
      } catch (refreshError) {
        const authStore = useAuthStore()
        authStore.clearAuth()
        window.location.href = '/login'
        return Promise.reject(refreshError)
      }
    }
    return Promise.reject(error)
  }
)
```

### 路由守卫

```typescript
// router/guards.ts
router.beforeEach(async (to, from, next) => {
  const authStore = useAuthStore()

  if (to.meta.requiresAuth) {
    if (!authStore.isAuthenticated) {
      try {
        await authStore.checkAuth()
        next()
      } catch {
        return next('/login')
      }
    }

    if (to.meta.requiresAdmin && !authStore.isAdmin) {
      return next('/403')
    }
  }

  if (to.path === '/login' && authStore.isAuthenticated) {
    return next('/admin')
  }

  next()
})
```

### checkAuth 实现

```typescript
async function checkAuth() {
  try {
    const res = await api.post('/api/auth/refresh', {}, {
      withCredentials: true,
    })
    const { access_token, user } = res.data.data
    setAuth(access_token, user)
    return true
  } catch {
    clearAuth()
    return false
  }
}
```

### 登出实现

```typescript
async function logout() {
  try {
    await api.post('/api/auth/logout', {}, {
      withCredentials: true,
    })
  } finally {
    clearAuth()
    router.push('/login')
  }
}
```

## 4.5 前端状态持久化策略

| 状态 | 存储方式 | 说明 |
|------|----------|------|
| Access Token | 内存（变量） | 不持久化，刷新页面丢失 |
| User Info | 内存（变量） | 不持久化，配合 Access Token |
| 路由状态 | Vue Router | URL 驱动，无需持久化 |

**设计原则**：Access Token 不持久化到 localStorage/sessionStorage，防止 XSS 攻击。
