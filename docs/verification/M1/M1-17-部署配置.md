# M1-17 - 部署配置

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m1-17-部署配置)
> 涉及 Spec: [07-部署配置.md](../../spec/07-部署配置.md)

## 验证目标

Docker 打包和 Nginx 配置正确，前后端可部署访问。

## 测试用例

### TC-01: Dockerfile 验证

**前置条件：**
- 后端项目结构完整

**测试步骤：**
1. 检查 `backend/Dockerfile` 存在
2. 检查内容

**预期结果：**
- Dockerfile 存在
- 基于 JDK 21 镜像
- 包含 Maven 构建步骤

### TC-02: 后端 Docker 构建

**前置条件：**
- TC-01 已通过
- Docker 环境正常

**测试步骤：**
1. 在 backend 目录执行 `docker build -t personal-website-backend .`
2. 等待构建完成

**预期结果：**
- 构建成功
- 镜像生成

### TC-03: Nginx 配置验证

**前置条件：**
- 前端项目结构完整

**测试步骤：**
1. 检查 nginx.conf 存在
2. 检查配置内容

**预期结果：**
- 配置前端静态文件服务
- 配置 API 反向代理到后端

### TC-04: docker-compose.yml 完整配置

**前置条件：**
- TC-01、TC-03 已通过

**测试步骤：**
1. 检查完整的 docker-compose.yml（包含后端、前端、Nginx）
2. 验证配置

**预期结果：**
- 包含 postgres、backend、frontend、nginx 服务
- 端口映射正确

### TC-05: 全量 Docker Compose 启动

**前置条件：**
- TC-04 已通过

**测试步骤：**
1. 执行 `docker-compose up -d`
2. 等待所有服务启动
3. 验证服务状态

**预期结果：**
- 所有容器运行中
- 前端可通过 Nginx 访问
- API 正常响应

### TC-06: 生产环境访问

**前置条件：**
- TC-05 已通过

**测试步骤：**
1. 通过 Nginx 访问前端 `http://localhost`
2. 验证页面正常显示

**预期结果：**
- 前端页面正常加载
- API 请求正常
