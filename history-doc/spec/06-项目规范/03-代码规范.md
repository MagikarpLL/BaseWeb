# 个人网站项目 - 代码规范

## 一、代码风格

### 1.1 后端 Java 代码风格

#### 自动化工具

- **Spotless** - 代码格式化 (基于 Google Java Format)
- **Checkstyle** - 代码质量检查

#### 格式化规则
- 使用 Google Java Format (AOSP 风格)
- 缩进: 4 空格
- 行宽: 120 字符 (Spotless 自动处理)
- 启用 `reflowLongStrings` 拆分超长字符串

#### 使用方式

```bash
# 格式化代码 (自动修复)
cd backend
mvn spotless:apply

# 检查格式 (不修改)
mvn spotless:check

# 运行 Checkstyle
mvn checkstyle:check
```

#### Maven 绑定 (自动执行)

| Phase | Plugin | 说明 |
|-------|--------|------|
| validate | checkstyle:check | Checkstyle 检查 |
| compile | spotless:apply | 格式化主代码 |

#### 命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `BlogPostService` |
| 方法名 | camelCase | `getPostBySlug` |
| 变量名 | camelCase | `postList`, `isValid` |
| 常量 | UPPER_SNAKE_CASE | `MAX_PAGE_SIZE` |
| 包名 | lowercase | `com.example.personalwebsite` |

#### 类结构顺序

```java
public class XxxServiceImpl implements XxxService {

    // 1. 日志 logger
    private static final Logger log = LoggerFactory.getLogger(XxxServiceImpl.class);

    // 2. 依赖注入
    @Autowired
    private XxxMapper xxxMapper;

    // 3. 公共方法 (实现接口的方法)

    // 4. 私有方法

}
```

#### 注解使用

```java

@RestController
@RequestMapping("/api/blog/posts")
@RequiredArgsConstructor

    // 参数校验

        // ...

        // ...

```

#### 格式化规则

| 规则 | 配置 |

```typescript
// 接口命名: I 前缀或 PascalCase
```

## 二、Git 提交规范

### 2.1 Commit Message 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 2.2 Type 类型

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档变更 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具变更 |

### 2.3 Scope 范围

| Scope | 说明 |
|-------|------|
| `backend` | 后端代码 |
| `frontend` | 前端代码 |
| `db` | 数据库相关 |
| `config` | 配置变更 |
| `deps` | 依赖更新 |
| `all` | 涉及全部 |

### 2.4 示例

```
feat(backend): 添加 JWT 认证功能

实现 Access Token 和 Refresh Token 分离
- Access Token 15 分钟有效期
- Refresh Token 7 天，存储在 HTTP-only Cookie

Closes #12
```

```

当 totalPages 为 0 时显示错误页码的问题
- 添加 totalPages 的边界检查
- 分页组件增加空状态处理

Closes #15
```

```

- 添加目录结构说明
```

## 五、异常处理规范

### 5.1 后端异常处理

```java
// 自定义业务异常
public class BusinessException extends RuntimeException {
    private final int code;

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }
}

// 在 Controller 中抛出
@ExceptionHandler(BusinessException.class)
public ApiResponse<?> handleBusiness(BusinessException ex) {
    return ApiResponse.error(ex.getCode(), ex.getMessage());
}
```

### 5.2 前端错误处理

```typescript
// api/axios.ts
api.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      // Token 过期，尝试刷新
      // ...
    }

    if (error.response?.status === 403) {
      ElMessage.error('无权限访问')
    }

    if (error.response?.status === 500) {
      ElMessage.error('服务器错误')
    }

    return Promise.reject(error)
  }
)
```

## 六、日志规范

### 6.1 后端日志

```java
@Slf4j
@Service
public class PostServiceImpl {

    public BlogPost getPostBySlug(String slug) {
        log.info("查询文章, slug={}", slug);

        BlogPost post = postMapper.findBySlug(slug);
        if (post == null) {
            log.warn("文章不存在, slug={}", slug);
            throw new ResourceNotFoundException("文章不存在");
        }

        // 增加阅读量
        postMapper.incrementReadCount(post.getId());
        log.info("文章查询成功, id={}", post.getId());

        return post;
    }
}
```

### 6.2 日志级别使用

| 级别 | 使用场景 |
|------|----------|
| DEBUG | 开发调试信息 |
| INFO | 正常业务流程 |
| WARN | 警告信息（可恢复的错误） |
| ERROR | 错误信息（需处理） |

## 七、安全规范

### 7.1 后端安全

- 所有用户输入必须校验
- SQL 参数使用 MyBatis 绑定变量 `#{}`，禁止 `${}`
- 敏感配置放环境变量
- 接口添加防爬限制（可选）

### 7.2 前端安全

- Token 不存 localStorage（存内存）
- 敏感操作需二次验证
- XSS 过滤（Vue 默认转义）
- CSP 配置（Nginx）
