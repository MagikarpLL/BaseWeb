# M1-05 - 统一响应封装 + 全局异常处理

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m1-05-统一响应封装-全局异常处理)
> 涉及 Spec: [02-API契约.md](../../spec/02-API契约.md), [08-错误码设计.md](../../spec/08-错误码设计.md)

## 验证目标

所有 API 返回统一格式，全局异常处理正常。

## 测试用例

### TC-01: 统一响应格式验证

**前置条件：**
- 后端服务已启动

**测试步骤：**
1. 访问任意 API（如 `/api/public/home`）
2. 验证响应格式

**预期结果：**
- 响应格式为 `{"code": 200, "message": "Success", "data": {...}}`

### TC-02: 成功响应格式

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 访问公开 API
2. 验证 data 字段包含数据

**预期结果：**
- code 为 200
- message 为 "Success" 或 null
- data 为实际数据

### TC-03: 错误响应格式

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 访问不存在的资源 `/api/admin/not-exist`
2. 验证错误响应

**预期结果：**
- code 非 200
- message 包含错误描述
- data 为 null

### TC-04: 全局异常处理

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 触发一个未处理的异常场景
2. 验证响应

**预期结果：**
- 返回 500 错误
- 响应格式保持统一

### TC-05: 参数校验异常

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 发送缺少必需参数的请求
2. 验证响应

**预期结果：**
- 返回 400 错误
- message 提示参数错误

## 验收标准

- [ ] TC-01: 统一响应格式验证通过
- [ ] TC-02: 成功响应格式通过
- [ ] TC-03: 错误响应格式通过
- [ ] TC-04: 全局异常处理通过
- [ ] TC-05: 参数校验异常通过
- [ ] 所有测试用例通过
