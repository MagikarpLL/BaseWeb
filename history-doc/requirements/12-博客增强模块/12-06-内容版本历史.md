# 12-06-内容版本历史

## 12.6.1 功能概述

每次发布文章时自动保存内容快照，支持版本对比和还原，帮助管理员管理内容变更历史。

## 12.6.3 功能详情

### 12.6.3.1 快照保存规则

|| 事件 | 是否保存快照 |
||------|-------------|
| 点击"发布"按钮 | ✅ 保存 |
| 点击"保存草稿" | ❌ 不保存 |
| 文章状态变更（草稿→发布） | ✅ 保存 |

### 12.6.3.2 版本保留策略

|| 项目 | 说明 |
||------|------|
| 最大版本数 | 10 个 |
| 超出的处理 | 删除最早的版本 |
| 版本排序 | 按创建时间倒序（最新在前） |

### 12.6.3.3 版本对比

|| 项目 | 说明 |
||------|------|
| 对比方式 | 左右分栏对比 |
| 差异标注 | 新增行绿色，删除行红色 |
| 版本选择 | 选择任意两个版本对比 |

### 12.6.3.4 版本还原

|| 项目 | 说明 |
||------|------|
| 还原行为 | 将指定版本的内容恢复到编辑器 |
| 保存机制 | 还原后创建新版本，不覆盖历史 |
| 确认流程 | 还原前需二次确认 |

## 12.6.4 验收标准

- [ ] 每次发布自动保存快照
- [ ] 保存草稿不保存快照
- [ ] 版本列表按时间倒序
- [ ] 最多保留 10 个版本
- [ ] 超出的版本自动删除最早的
- [ ] 可以查看所有历史版本
- [ ] 可以对比任意两个版本的差异
- [ ] 差异以颜色区分（新增/删除）
- [ ] 可以还原到指定版本
- [ ] 还原后创建新版本（不覆盖历史）
- [ ] 还原前需二次确认
