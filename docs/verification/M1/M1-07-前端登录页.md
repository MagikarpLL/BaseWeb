# M1-07 - 前端登录页 + Auth Store + 路由守卫

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m1-07-前端登录页-auth-store-路由守卫)
> 涉及 Spec: [09-前端路由设计.md](../../spec/09-前端路由设计.md)

## 验证目标

前端登录页正常显示，Auth Store 状态管理正常，路由守卫工作正常。

## 测试用例

### TC-01: 登录页访问

**前置条件：**
- 前端服务已启动

**测试步骤：**
1. 访问 `/login`
2. 验证页面加载

**预期结果：**
- 登录页面正常显示
- 包含用户名、密码输入框和登录按钮

### TC-02: 登录表单验证

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 不填写用户名点击登录
2. 验证提示

**预期结果：**
- 提示用户名不能为空

### TC-03: 登录成功

**前置条件：**
- TC-01 已通过
- 存在有效账户

**测试步骤：**
1. 填写正确的用户名密码
2. 点击登录
3. 验证跳转

**预期结果：**
- 登录成功 toast
- 跳转到 `/admin`

### TC-04: 登录失败

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 填写错误的密码
2. 点击登录
3. 验证提示

**预期结果：**
- 提示登录失败
- 停留在登录页

### TC-05: Auth Store 状态

**前置条件：**
- TC-03 已通过

**测试步骤：**
1. 登录后检查 Pinia Auth Store 状态
2. 验证 token 和 user 信息

**预期结果：**
- token 已存储
- user 信息正确

### TC-06: 路由守卫 - 未登录访问后台

**前置条件：**
- 未登录状态

**测试步骤：**
1. 直接访问 `/admin`
2. 验证跳转

**预期结果：**
- 跳转到 `/login`

### TC-07: 路由守卫 - 已登录访问后台

**前置条件：**
- 已登录状态

**测试步骤：**
1. 直接访问 `/admin`
2. 验证正常显示

**预期结果：**
- 正常显示管理后台
