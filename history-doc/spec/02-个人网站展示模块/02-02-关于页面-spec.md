# 02-02-关于页面

> 对应 Requirement: [02-02-关于页面.md](../requirements/02-个人网站展示模块/02-02-关于页面.md)

## 2.2.1 设计原则

| 原则 | 说明 |
|------|------|
| 内容优先 | 关于页内容完整呈现，无限滚动体验 |
| 数据驱动 | 所有内容来自后端 API 配置 |
| 可维护性 | 结构化存储 skills/experiences/projects，支持增删改 |

## 2.2.3 页面渲染规格

### Hero 区域

| 字段 | 渲染规则 |
|------|----------|
| 头像 | `200x200px` 圆形，hover 放大 `1.05` 倍 |
| 姓名 | 字号 `2.5rem`，加粗 |
| 职位 | 字号 `1.25rem`，颜色 `#666` |
| 位置 | 图标 + 文本，格式 `📍 上海` |

### 个人介绍

| 字段 | 渲染规则 |
|------|----------|
| aboutContent | Markdown 渲染，支持 `h2`/`h3`/`p`/`ul`/`ol`/`blockquote` |
| 位置 | 文本，紧跟在 Hero 下方 |

### 技能展示

| 分类 | 渲染规则 |
|------|----------|
| frontend | 进度条显示，`level` 值为百分比（0-100） |
| backend | 进度条显示 |
| database | 进度条显示 |
| tools | 标签云展示 |

**进度条样式**：
- 容器：`200px` 宽，`8px` 高，圆角 `4px`
- 填充：从左到右渐变 `background: linear-gradient(90deg, #667eea, #764ba2)`
- 动画：页面加载时从 0% 动画展开到目标值

### 经历时间线

| 类型 | 布局规则 |
|------|----------|
| 教育背景 | 左侧时间线，右侧内容卡片 |
| 工作经历 | 左侧时间线，右侧内容卡片 |

**时间线样式**：
- 竖线：`2px` 宽，颜色 `#e0e0e0`
- 节点：`12px` 圆形，激活状态颜色 `#667eea`
- 时间范围：字号 `0.875rem`，颜色 `#999`

### 项目展示

| 字段 | 渲染规则 |
|------|----------|
| 卡片布局 | 网格布局，`3` 列（桌面）/ `2` 列（平板）/ `1` 列（手机） |
| 图片 | 高度 `200px`，`object-fit: cover` |
| 项目名称 | 字号 `1.25rem`，加粗 |
| 技术栈 | 标签样式，背景 `#f0f0f0` |
| 链接 | 图标按钮，hover 显示下划线 |

**显示规则**：只显示 `featured=true` 的项目，最多 `6` 个。

---

## 2.2.4 关键逻辑

### 技能分类筛选

```
skills → groupBy(category) → {
  frontend: [...],
  backend: [...],
  database: [...],
  tools: [...]
}
```

### 项目排序

```
projects.filter(featured=true) → sortBy(featured desc, created_at desc) → limit(6)
```

## 2.2.5 验收标准

- [ ] 页面正确显示所有个人信息（头像/姓名/职位/位置/介绍）
- [ ] 技能以进度条和标签云方式可视化展示
- [ ] 教育背景和工作经历以时间线形式展示
- [ ] 项目以卡片形式展示，最多 6 个精选项目
- [ ] 项目图片/链接正确显示
- [ ] 响应式布局：移动端单列显示
- [ ] SEO meta 标签正确设置（title/description/og:type=profile）
- [ ] 页面加载性能满足 Core Web Vitals
