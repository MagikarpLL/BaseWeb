## 9.3 路由守卫

### 全局前置守卫

```typescript
router.beforeEach(async (to, from) => {
  const authStore = useAuthStore()

  // 设置页面标题
  if (to.meta.title) {
    document.title = `${to.meta.title} - ${import.meta.env.VITE_SITE_NAME || '个人网站'}`
  }

  // 需要认证的页面
  if (to.matched.some(record => record.meta.requiresAuth)) {
    if (!authStore.isLoggedIn) {
      return {
        name: 'login',
        query: { redirect: to.fullPath }
      }
    }
  }

  // 已登录用户访问登录页，重定向到管理后台
  if (to.meta.guest && authStore.isLoggedIn) {
    return { name: 'admin-dashboard' }
  }
})
```

### 守卫规则汇总

| 目标路由 | 未登录 | 已登录 |
|----------|--------|--------|
| 前台页面 (`/`, `/blog`, `/about`, `/tools`) | 正常访问 | 正常访问 |
| `/login` | 正常访问 | 重定向 `/admin` |
| `/admin/*` | 重定向 `/login?redirect=原路径` | 正常访问 |

## 9.4 懒加载策略

| 策略 | 适用范围 | 说明 |
|------|----------|------|
| 路由级懒加载 | 所有页面组件 | `() => import('@/views/xxx.vue')` |
| 组件级懒加载 | 重型组件（Markdown 编辑器、图表） | `defineAsyncComponent()` |
| 图片懒加载 | 文章内容中的图片 | `loading="lazy"` + Intersection Observer |

### 预加载策略

| 场景 | 策略 |
|------|------|
| 用户 hover 导航链接 | 预加载目标路由组件（`router.prefetch()`） |
| 管理后台侧边栏 hover | 预加载目标管理页面 |

## 9.5 路由元信息 (meta) 规范

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| title | string | 是 | 页面标题，用于 `document.title` |
| requiresAuth | boolean | 否 | 是否需要登录，默认 false |
| guest | boolean | 否 | 仅未登录可访问（如登录页） |
| activeMenu | string | 否 | 管理后台侧边栏高亮项（处理编辑页等子页面） |
| breadcrumb | Array<{name, path}> | 否 | 面包屑导航配置 |

### 示例：文章编辑页的 meta

```typescript
{
  path: 'posts/:id/edit',
  name: 'admin-post-edit',
  component: () => import('@/views/admin/PostEditView.vue'),
  meta: {
    title: '编辑文章',
    activeMenu: 'admin-posts',
    breadcrumb: [
      { name: '仪表盘', path: '/admin' },
      { name: '文章管理', path: '/admin/posts' },
      { name: '编辑文章' }
    ]
  }
}
```

## 9.6 404 处理

- 前台 catch-all 路由 `/:pathMatch(.*)*` 匹配所有未定义路径
- 管理后台未匹配路径也由前台 catch-all 捕获（因为 `/admin/xxx` 不匹配时，走外层 `/:pathMatch`）
- 文章详情页 `/blog/:slug` 中 slug 无效时，由组件内逻辑判断后调用 `router.replace({ name: 'not-found' })`
