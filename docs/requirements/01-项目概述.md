# 01-项目概述

## 1.1 项目背景

### 项目名称
个人网站 + 博客系统

### 项目定位
一个集个人展示、博客发布、小工具集于一体的综合性个人网站。

### 核心价值
- **个人品牌**：展示个人技能、项目经历、专业领域
- **知识沉淀**：通过博客记录学习历程、技术分享
- **工具集**：提供一些实用的在线小工具

### 为什么选择 SPA 架构
根据用户需求分析：
- 首页需要嵌入多个小工具（Dashboard 风格）
- 工具集模块需要动态交互和实时查询
- 博客和工具集之间可能有复杂的交互逻辑
- Vue SPA 更适合这类"应用型"个人网站

## 1.4 非功能需求

### 性能需求
- 首屏加载时间 < 3s（4G 网络）
- LCP < 2.5s
- INP < 200ms
- CLS < 0.1

### 安全需求
- JWT Access Token 有效期 15 分钟
- Refresh Token 有效期 7 天，存储在 HTTP-only Cookie
- 敏感操作需要重新验证
- SQL 注入防护
- XSS 防护

### 兼容性需求
- 浏览器：Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- 移动端：响应式布局，支持 iOS Safari 和 Android Chrome

### SEO 需求
- 所有公开页面支持 meta 标签
- 生成 sitemap.xml 和 robots.txt
- 支持 JSON-LD 结构化数据
- 博客详情页预渲染

## 1.5 约束与假设

### 约束
- 单体架构，不拆分微服务
- 数据库使用 PostgreSQL，不使用其他存储
- 数据库部署使用 Docker 模式
- 前端不引入 SSR/Nuxt，保持 Vue SPA
- 图片存储在本地服务器（不使用图床或 OSS）

### 假设
- 访客数量级：日均 PV < 10000
- 文章数量：< 1000 篇
- 工具集功能不超过 10 个
- 单管理员使用场景
- 评论数量：平均每篇文章 < 50 条评论
- 需要评论审核功能（防止垃圾评论）
