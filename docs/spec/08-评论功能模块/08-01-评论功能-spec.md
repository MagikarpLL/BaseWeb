| 原则 | 说明 |
|------|------|
| 审核机制 | 评论需审核后才能显示，防止垃圾信息 |

### 提交评论

**接口**: `POST /api/blog/posts/:slug/comments`

**请求**:
```json
{
  "authorName": "访客",
  "authorEmail": "visitor@example.com",
  "content": "写得很好！",
  "parentId": null,
  "captcha": "abc123"
}
```

**响应**:
```json
{
  "code": 200,
  "message": "评论提交成功，等待审核"
}
```

**接口**: `PUT /api/admin/comments/:id/status`


## 8.1.3 页面渲染规格

### 文章评论区

| 元素 | 渲染规则 |
|------|----------|
| 标题 | `评论 ({n})`，n 为 approved 评论数 |
| 评论表单 | 昵称 / 邮箱 / 评论内容 / 验证码 / 提交按钮 |
| 评论列表 | 仅显示 approved 状态，按时间正序 |
| 嵌套回复 | 最多 2 层，缩进 `24px` |

**评论表单字段**：
| 字段 | 类型 | 规则 |
|------|------|------|
| 昵称 | 输入框 | 必填，2-20 字符 |
| 邮箱 | 输入框 | 必填，有效邮箱格式 |
| 评论内容 | 文本域 | 必填，5-2000 字符，支持 Markdown |
| 验证码 | 输入框 + 刷新按钮 | 必填，4 位字符 |

**评论卡片样式**：
- 头像：`48x48px` 圆形，Gravatar
- 昵称：加粗，颜色 `#333`
- 时间：相对时间，如"3 小时前"
- 内容：正常字号，颜色 `#666`
- 回复按钮：hover 显示

### 管理后台评论列表

| 元素 | 渲染规则 |
|------|----------|
| Tab 切换 | 待审核(n) / 已通过(n) / 已拒绝(n) |
| 筛选 | 状态下拉 + 文章下拉 + 搜索框 |
| 列表 | 评论内容摘要 + 作者 + 来源文章 + 时间 + 操作 |
| 操作按钮 | 批准 / 拒绝 / 删除 |

**评论卡片样式**：
- 背景：白色，1px 边框
- 状态标签：待审核（黄色）/ 已通过（绿色）/ 已拒绝（红色）
- 操作按钮：文本按钮，hover 变色

## 8.1.4 关键逻辑

### 评论显示规则

```typescript
// 只显示 approved 状态
const visibleComments = comments.filter(c => c.status === 'approved')

// 按时间正序排列
visibleComments.sort((a, b) => new Date(a.createdAt) - new Date(b.createdAt))

// 构建嵌套结构（最多 2 层）
function buildNestedComments(comments: Comment[]): NestedComment[] {
  return comments.map(comment => ({
    ...comment,
    replies: comment.replies
      .filter(r => r.status === 'approved')
      .map(reply => ({
        ...reply,
        replies: []  // 第二层不显示回复
      }))
  }))
}
```

### 验证码验证

```typescript
// 验证码生成（后端）
function generateCaptcha() {
  const chars = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789'
  const captcha = Array(4).fill('').map(() => chars[Math.floor(Math.random() * chars.length)]).join('')
  // 存储到 Session，有效期 10 分钟
  return captcha
}

// 验证码校验（后端）
function validateCaptcha(input: string, sessionCaptcha: string): boolean {
  return input.toUpperCase() === sessionCaptcha.toUpperCase() && !isExpired(sessionCaptcha)
}
```

### @ 提及处理

```typescript
// 评论内容中的 @ 提及转换为链接
function parseMentions(content: string): string {
  return content.replace(/@(\w+)/g, '<a href="/user/$1">@$1</a>')
}
```

## 8.1.5 状态流转

```
[访客提交] → pending → [管理员批准] → approved
                         → [管理员拒绝] → rejected

[管理员回复] → 直接创建 approved 状态的回复
```

---

## 8.1.6 验收标准

- [ ] 访客可以提交评论（填写昵称/邮箱/内容/验证码）
- [ ] 评论提交后状态为 pending，不立即显示
- [ ] 提交成功显示"评论等待审核"提示
- [ ] 评论列表正确显示 approved 评论（嵌套结构，最多 2 层）
- [ ] 管理员可以查看所有评论（待审核/已通过/已拒绝）
- [ ] 管理员可以批准评论（变为 approved 状态）
- [ ] 管理员可以拒绝评论（变为 rejected 状态）
- [ ] 管理员可以回复评论（直接 approved）
- [ ] 管理员可以删除评论（永久删除）
- [ ] 评论列表支持分页（每页 10 条）
- [ ] 验证码防止机器人（4 位字符，10 分钟有效）
- [ ] 评论内容支持 Markdown（粗体/斜体/代码/链接）
- [ ] @ 提及显示为可点击链接
