# 01-迭代计划

> 本文档定义项目的迭代划分、MVP 边界、任务拆分和依赖关系，是 Harness 循环中 Plan 层的核心文档。

## 1.1 迭代总览

|| 迭代 | 名称 | 核心目标 | 预估周期 |
|------|------|----------|----------|
| M1 | 基础框架 + 博客核心 | 跑通"登录→写文章→看文章"主流程 | 3 周 |
| M2 | 内容管理补全 | 分类标签、配置、关于页、工具集 | 2 周 |
| M3 | 互动功能 | 评论系统、访问统计 | 2 周 |
| M4 | 精益打磨 | SEO、性能优化、错误体验 | 1 周 |

### MVP 定义

**MVP = M1 完成后的状态**，此时应具备：
- 管理员可登录后台
- 管理员可创建/编辑/发布/删除文章
- 访客可在前台浏览文章列表和文章详情
- 首页展示个人信息和最新文章
- 前后端部署可访问

## 1.7 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Markdown 编辑器选型影响开发效率 | 中 | M1 阶段先使用成熟库（如 md-editor-v3），避免自研 |
| SPA 的 SEO 限制 | 中 | M4 阶段通过预渲染 + sitemap 缓解，不引入 SSR |
| 图片存储空间增长 | 低 | 单管理员场景下增长有限，后期可迁移 OSS |
| 工具集页面扩展性 | 低 | 采用路由配置式加载，新工具只需加配置和组件 |
| 翻译遗漏 | 中 | 逐个页面检查，制定完整翻译清单 |
