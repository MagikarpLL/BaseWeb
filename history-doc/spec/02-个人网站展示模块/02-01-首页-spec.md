## 2.1.1 设计原则

| 原则 | 说明 |
|------|------|
| 快速加载 | 首屏 3 秒内完成加载，Hero 区域优先渲染 |
| 数据驱动 | 首页内容来自后端 API，非硬编码 |
| 优雅降级 | API 失败时显示静态内容或空状态 |

## 2.1.3 页面渲染规格

### Hero 区域

| 字段 | 渲染规则 |
|------|----------|
| 头像 | `150x150px` 圆形，`object-fit: cover`，加载失败显示默认占位图 |
| 姓名 | 字号 `2rem`，颜色 `#333` |
| 职位 | 字号 `1.25rem`，颜色 `#666` |
| 个人简介 | 字号 `1rem`，最多显示 2 行，超出省略 |
| 社交链接 | 图标 `24x24px`，hover 放大至 `28x28px`，新窗口打开 |

### 最新文章区域

| 字段 | 渲染规则 |
|------|----------|
| 标题 | 最多 `50` 字符，超出截断 + `...` |
| 摘要 | 最多 `150` 字符，超出截断 |
| 日期 | 格式 `YYYY-MM-DD` |
| 阅读量 | 数字格式化，如 `1,234` |
| 卡片高度 | `flex` 布局，等高排列 |

**显示规则**：取 `status='published'` 且 `published_at` 最近 3 篇。

### 精选工具区域

| 字段 | 渲染规则 |
|------|----------|
| 工具名称 | 最多 `20` 字符 |
| 图标 | Font Awesome，尺寸 `48x48px` |
| 描述 | 最多 `80` 字符 |
| 卡片 | 固定 `200x180px`，hover 上移 `4px` + 阴影加深 |

**显示规则**：取 `is_featured=true` 的 3 个工具。

## 2.1.4 关键逻辑

### 加载顺序

```
1. 页面骨架渲染（Header + Footer 立即显示）
2. 并行请求 /api/public/home
3. Hero 区域渲染（等待 profile 数据）
4. 最新文章 + 精选工具（并行渲染）
```

### 空状态处理

| 场景 | 处理方式 |
|------|----------|
| 无文章 | 显示"暂无文章"文案 + 跳转博客按钮 |
| 无工具 | 隐藏精选工具区域 |
| API 失败 | 显示 Hero + Toast "数据加载失败，请刷新重试" |

## 2.1.5 验收标准

- [ ] 首页在 3 秒内完成加载（LCP < 2.5s）
- [ ] Hero 区域正确显示用户信息（头像/姓名/职位/简介/社交链接）
- [ ] 最新文章区域正确显示最近 3 篇文章
- [ ] 精选工具区域正确显示 3 个工具
- [ ] 点击文章卡片跳转 `/blog/{slug}`
- [ ] 点击工具卡片跳转 `/tools/{toolPath}`
- [ ] 社交链接点击新窗口打开
- [ ] SEO meta 标签正确设置（title/description/og:*）
- [ ] 响应式布局：移动端文章卡片单列显示
