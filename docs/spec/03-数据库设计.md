## 3.3 数据表定义

### user（用户表）

```sql
CREATE TABLE "user" (
    id              SERIAL PRIMARY KEY,
    username        VARCHAR(50) NOT NULL UNIQUE,
    password        VARCHAR(255) NOT NULL,
    role            VARCHAR(20) NOT NULL DEFAULT 'ADMIN',
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    last_login_at   TIMESTAMP,
    login_attempts  INTEGER NOT NULL DEFAULT 0,
    locked_until    TIMESTAMP
);

CREATE INDEX idx_user_username ON "user"(username);
```

### category（分类表）

```sql
CREATE TABLE category (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(20) NOT NULL,
    slug            VARCHAR(50) NOT NULL UNIQUE,
    description     VARCHAR(200),
    sort            INTEGER NOT NULL DEFAULT 0,
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_category_slug ON category(slug);
CREATE INDEX idx_category_sort ON category(sort);
```

### tag（标签表）

```sql
CREATE TABLE tag (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(15) NOT NULL,
    slug            VARCHAR(50) NOT NULL UNIQUE,
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_tag_slug ON tag(slug);
```

### blog_post（文章表）

```sql
CREATE TABLE blog_post (
    id              SERIAL PRIMARY KEY,
    title           VARCHAR(200) NOT NULL,
    slug            VARCHAR(100) NOT NULL UNIQUE,
    excerpt         VARCHAR(500),
    content         TEXT NOT NULL,
    cover_image     VARCHAR(500),
    status          VARCHAR(20) NOT NULL DEFAULT 'draft',
    reading_time    INTEGER NOT NULL DEFAULT 0,
    read_count      INTEGER NOT NULL DEFAULT 0,

    author_id       INTEGER NOT NULL REFERENCES "user"(id),
    category_id     INTEGER REFERENCES category(id),

    published_at    TIMESTAMP,
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    deleted_at      TIMESTAMP
);

CREATE INDEX idx_blog_post_slug ON blog_post(slug);
CREATE INDEX idx_blog_post_status ON blog_post(status);
CREATE INDEX idx_blog_post_published_at ON blog_post(published_at);
CREATE INDEX idx_blog_post_category ON blog_post(category_id);
CREATE INDEX idx_blog_post_author ON blog_post(author_id);
CREATE INDEX idx_blog_post_deleted_at ON blog_post(deleted_at);
```

### blog_post_tag（文章标签关联表）

```sql
CREATE TABLE blog_post_tag (
    post_id     INTEGER NOT NULL REFERENCES blog_post(id) ON DELETE CASCADE,
    tag_id      INTEGER NOT NULL REFERENCES tag(id) ON DELETE CASCADE,
    PRIMARY KEY (post_id, tag_id)
);

CREATE INDEX idx_blog_post_tag_tag ON blog_post_tag(tag_id);
```

### comment（评论表）

```sql
CREATE TABLE comment (
    id              SERIAL PRIMARY KEY,
    post_id         INTEGER NOT NULL REFERENCES blog_post(id) ON DELETE CASCADE,
    author_name     VARCHAR(50) NOT NULL,
    author_email    VARCHAR(100) NOT NULL,
    content         TEXT NOT NULL,
    parent_id       INTEGER REFERENCES comment(id) ON DELETE CASCADE,
    status          VARCHAR(20) NOT NULL DEFAULT 'pending',
    ip_address      VARCHAR(45),
    user_agent      VARCHAR(500),
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_comment_post_id ON comment(post_id);
CREATE INDEX idx_comment_status ON comment(status);
CREATE INDEX idx_comment_parent_id ON comment(parent_id);
CREATE INDEX idx_comment_created_at ON comment(created_at);
```

### site_settings（网站配置表）

```sql
CREATE TABLE site_settings (
    id              SERIAL PRIMARY KEY,
    key             VARCHAR(100) NOT NULL UNIQUE,
    value           JSONB NOT NULL,
    type            VARCHAR(20) NOT NULL DEFAULT 'string',
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_site_settings_key ON site_settings(key);
```

### page_view（页面访问表 - 按小时聚合）

```sql
CREATE TABLE page_view (
    id              SERIAL PRIMARY KEY,
    page_url        VARCHAR(500) NOT NULL,
    pv_count        INTEGER NOT NULL DEFAULT 0,
    uv_count        INTEGER NOT NULL DEFAULT 0,
    date            DATE NOT NULL,
    hour            INTEGER NOT NULL CHECK (hour >= 0 AND hour <= 23),
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    UNIQUE (page_url, date, hour)
);

CREATE INDEX idx_page_view_date ON page_view(date);
CREATE INDEX idx_page_view_page_url ON page_view(page_url);
```

### unique_visitor（独立访客表）

```sql
CREATE TABLE unique_visitor (
    id              SERIAL PRIMARY KEY,
    visitor_id      VARCHAR(100) NOT NULL UNIQUE,
    first_visit_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    last_visit_at  TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    total_pv       INTEGER NOT NULL DEFAULT 0
);

CREATE INDEX idx_unique_visitor_visitor_id ON unique_visitor(visitor_id);
CREATE INDEX idx_unique_visitor_last_visit ON unique_visitor(last_visit_at);
```

### daily_stats（每日汇总表）

```sql
CREATE TABLE daily_stats (
    id              SERIAL PRIMARY KEY,
    date            DATE NOT NULL UNIQUE,
    total_pv        INTEGER NOT NULL DEFAULT 0,
    total_uv        INTEGER NOT NULL DEFAULT 0,
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_daily_stats_date ON daily_stats(date);
```

## 3.4 表关系说明

| 关系 | 说明 |
|------|------|
| User → BlogPost | 一对多，一个用户可以写多篇文章 |
| Category → BlogPost | 一对多，一个分类包含多篇文章 |
| Tag ↔ BlogPost | 多对多，通过 blog_post_tag 关联 |
| BlogPost → Comment | 一对多，一篇文章有多条评论 |
| Comment → Comment | 自关联，用于回复功能 |
| site_settings | KV 表存储各类配置 |

## 3.5 软删除设计

blog_post 表使用 `deleted_at` 字段实现软删除：
- `deleted_at IS NULL` → 文章存在
- `deleted_at IS NOT NULL` → 文章已删除

查询有效文章时需添加 `WHERE deleted_at IS NULL` 条件。

## 3.6 状态枚举

### 文章状态 (blog_post.status)

| 状态值 | 说明 |
|--------|------|
| `draft` | 草稿，不会公开显示 |
| `published` | 已发布，公开显示 |

### 评论状态 (comment.status)

| 状态值 | 说明 |
|--------|------|
| `pending` | 待审核 |
| `approved` | 已通过 |
| `rejected` | 已拒绝 |

### 用户角色 (user.role)

| 角色值 | 说明 |
|--------|------|
| `ADMIN` | 管理员，有管理权限 |
