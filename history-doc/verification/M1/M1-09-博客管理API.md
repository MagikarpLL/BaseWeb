# M1-09 - 博客管理 API

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m1-09-博客管理-api)
> 涉及 Spec: [02-API契约.md](../../spec/02-API契约.md)

## 验证目标

博客管理 API（CRUD + 发布/草稿）正常工作。

## 测试用例

### TC-01: 创建文章

**前置条件：**
- 管理员已登录

**测试步骤：**
1. POST `/api/admin/posts` 请求体 `{"title": "测试文章", "content": "# 测试\n\n内容", "status": "draft"}`
2. 验证返回

**预期结果：**
- 返回 200 和文章数据
- 包含文章 ID
- 状态为 draft

### TC-02: 文章列表

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. GET `/api/admin/posts`
2. 验证返回文章列表

**预期结果：**
- 返回文章数组
- 包含 title、status、created_at 等字段

### TC-03: 获取单篇文章

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. GET `/api/admin/posts/{id}`
2. 验证返回文章详情

**预期结果：**
- 返回完整文章数据
- 包含 content 字段

### TC-04: 更新文章

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. PUT `/api/admin/posts/{id}` 请求体 `{"title": "更新后的标题"}`
2. 验证返回更新后的数据

**预期结果：**
- 标题已更新

### TC-05: 发布文章

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. PUT `/api/admin/posts/{id}/publish`
2. 验证状态变更

**预期结果：**
- 状态变为 published
- published_at 已设置

### TC-06: 删除文章

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. DELETE `/api/admin/posts/{id}`
2. 验证返回 204
3. 尝试获取该文章

**预期结果：**
- 删除成功
- 获取返回 404

### TC-07: 草稿状态验证

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 创建状态为 draft 的文章
2. 访问前台 API `/api/blog/posts`
3. 验证草稿不显示

**预期结果：**
- 草稿文章不在前台列表中
