# M5-02 - 翻译文件创建 - 验证文档

## 任务概述
创建前台页面的英文和中文翻译文件

## 交付物
- `frontend/src/locales/en.json` - 英文翻译
- `frontend/src/locales/zh-CN.json` - 中文翻译

## 验收标准

1. ✅ en.json 包含所有前台页面英文翻译
2. ✅ zh-CN.json 包含所有前台页面中文翻译
3. ✅ 两个文件的 key 结构一致
4. ✅ 包含 nav, home, about, tools, blog, footer, common 等主要分类

## 测试用例

### 测试 1: 翻译文件结构验证

**步骤**:
1. 读取 `frontend/src/locales/en.json`
2. 读取 `frontend/src/locales/zh-CN.json`
3. 对比两个文件的 key 结构

**预期结果**:
- 两个文件的 key 完全一致
- 包含以下顶级 key：nav, home, about, tools, blog, blogDetail, footer, admin, common

### 测试 2: nav 分类翻译验证

**步骤**:
1. 检查 en.json 中 nav 部分是否包含：home, blog, tools, about, login, logout, dashboard
2. 检查 zh-CN.json 中对应翻译是否非空

**预期结果**:
- nav.home = "Home" / "首页"
- nav.blog = "Blog" / "博客"
- nav.tools = "Tools" / "工具"
- nav.about = "About" / "关于"
- nav.login = "Login" / "登录"
- nav.logout = "Logout" / "退出"
- nav.dashboard = "Dashboard" / "仪表盘"

### 测试 3: common 分类翻译验证

**步骤**:
1. 检查 common 部分是否包含：save, cancel, confirm, delete, edit, add, search, loading

**预期结果**:
- 所有 common key 都有对应翻译

### 测试 4: JSON 格式验证

**步骤**:
1. 使用 JSON.parse 验证两个文件都是有效的 JSON

**预期结果**:
- 两个文件都能被正确解析，无语法错误
