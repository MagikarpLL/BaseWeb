# M4-04 - 后端错误码体系完善

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m4-04-后端错误码体系完善)
> 涉及 Spec: [08-错误码设计.md](../../spec/08-错误码设计.md)

## 验证目标

API 错误返回规范化错误码，便于前端处理。

## 测试用例

### TC-01: 统一错误响应格式

**前置条件：**
- 无

**测试步骤：**
1. 触发一个 API 错误（如访问不存在的资源）
2. 验证响应格式

**预期结果：**
- 返回 JSON 格式 `{"code": xxx, "message": "xxx", "data": null}`

### TC-02: 错误码对应关系

**前置条件：**
- 无

**测试步骤：**
1. 触发不同类型的错误
2. 验证错误码

**预期结果：**
- 400: 参数错误
- 401: 未认证
- 403: 无权限
- 404: 资源不存在
- 500: 服务器错误

### TC-03: 全局异常处理

**前置条件：**
- TC-01 已通过

**测试步骤：**
1. 触发任意未处理的异常
2. 验证响应

**预期结果：**
- 异常被 GlobalExceptionHandler 捕获
- 返回规范化错误响应
