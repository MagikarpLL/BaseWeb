# M4-03 - 前端错误边界与提示优化

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m4-03-前端错误边界与提示优化)
> 涉及 Spec: [08-错误码设计.md](../../spec/08-错误码设计.md)

## 验证目标

前端网络错误/业务错误有友好提示，不崩溃。

## 测试用例

### TC-01: 网络错误提示

**前置条件：**
- 无网络连接或服务器不可用

**测试步骤：**
1. 刷新页面或触发 API 请求
2. 验证错误提示

**预期结果：**
- 显示友好错误提示（不是空白页或代码）
- 显示"网络错误"或类似文案

### TC-02: 业务错误提示

**前置条件：**
- 服务器返回业务错误

**测试步骤：**
1. 提交一个触发业务错误的请求（如重复提交）
2. 验证错误提示

**预期结果：**
- 显示服务器返回的错误消息
- 提示用户如何处理

### TC-03: 组件错误边界

**前置条件：**
- 无

**测试步骤：**
1. 手动触发一个组件渲染错误（通过 React DevTools 或注入错误）
2. 验证页面行为

**预期结果：**
- 错误边界捕获错误
- 显示备用 UI 或错误提示
- 页面其他部分正常显示

## 验收标准

- [ ] TC-01: 网络错误提示通过
- [ ] TC-02: 业务错误提示通过
- [ ] TC-03: 组件错误边界通过
- [ ] 所有测试用例通过
