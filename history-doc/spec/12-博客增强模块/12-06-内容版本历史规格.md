# 12-06-内容版本历史规格

> 对应 Requirement: [12-06-内容版本历史.md](../requirements/12-博客增强模块/12-06-内容版本历史.md)

## 12.6.1 概述 / 设计原则

|| 原则 | 说明 |
||------|------|
|| 发布时保存 | 只有发布操作才创建快照 |
|| 最多10个版本 | 超出时自动删除最早的 |
|| 不可变历史 | 历史版本永不修改，只可还原 |

## 12.6.2 具体规格

### 12.6.2.1 数据库模型

```sql
CREATE TABLE post_history (
    id BIGSERIAL PRIMARY KEY,
    post_id BIGINT REFERENCES blog_post(id) ON DELETE CASCADE,
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,
    excerpt TEXT,
    status VARCHAR(20) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by BIGINT REFERENCES users(id)
);

CREATE INDEX idx_post_history_post_id ON post_history(post_id);
CREATE INDEX idx_post_history_created_at ON post_history(created_at DESC);
```

### 12.6.2.2 保存时机

在 `BlogPostService.publishPost()` 中添加保存逻辑：

```java
@Override
public void publishPost(Long id) {
    BlogPost post = findById(id);
    post.setStatus("published");
    post.setPublishedAt(LocalDateTime.now());
    update(post);

    // 保存快照
    saveHistorySnapshot(post, "published");
}

private void saveHistorySnapshot(BlogPost post, String status) {
    PostHistory history = new PostHistory();
    history.setPostId(post.getId());
    history.setTitle(post.getTitle());
    history.setContent(post.getContent());
    history.setExcerpt(post.getExcerpt());
    history.setStatus(status);
    history.setCreatedBy(getCurrentUserId());

    postHistoryService.save(history);

    // 清理超过10个的版本
    cleanupOldVersions(post.getId());
}
```

### 12.6.2.3 版本清理

```java
private void cleanupOldVersions(Long postId) {
    List<PostHistory> versions = postHistoryService.findByPostId(postId);

    if (versions.size() > 10) {
        // 按创建时间升序排序，删除最早的
        List<PostHistory> toDelete = versions.stream()
            .sorted(Comparator.comparing(PostHistory::getCreatedAt))
            .limit(versions.size() - 10)
            .collect(toList());

        postHistoryService.deleteByIds(toDelete.stream().map(PostHistory::getId).collect(toList()));
    }
}
```

### 12.6.2.4 API 端点

#### GET /api/admin/posts/{id}/history

|| 接口 | 方法 | 说明 | 认证 |
||------|------|------|------|
|| `/api/admin/posts/{id}/history` | GET | 获取版本列表 | JWT (Admin) |

**响应格式**：
```json
{
  "code": 200,
  "data": [
    { "id": 3, "createdAt": "2026-05-29T14:30:00Z", "status": "published" },
    { "id": 2, "createdAt": "2026-05-28T10:20:00Z", "status": "published" },
    { "id": 1, "createdAt": "2026-05-27T09:15:00Z", "status": "published" }
  ]
}
```

#### GET /api/admin/posts/{id}/history/{historyId}

获取单版本详情，包含 title、content、excerpt。

#### GET /api/admin/posts/{id}/history/compare

**查询参数**：`from={historyId1}&to={historyId2}`

**响应格式**：
```json
{
  "code": 200,
  "data": {
    "from": { "id": 1, "content": "旧版本内容..." },
    "to": { "id": 2, "content": "新版本内容..." },
    "diff": [
      { "type": "equal", "lines": ["相同行..."] },
      { "type": "insert", "lines": ["新增行..."] },
      { "type": "delete", "lines": ["删除行..."] }
    ]
  }
}
```

#### POST /api/admin/posts/{id}/history/{historyId}/restore

还原到指定版本，会创建新版本而不是覆盖历史。

import com.github.difflib.DiffUtils;

List<String> originalLines = Arrays.asList(original.split("\n"));

## 12.6.3 验收标准

- [ ] 发布文章时自动保存快照
- [ ] 保存草稿不保存快照
- [ ] 版本列表按时间倒序
- [ ] 最多保留 10 个版本
- [ ] 超出时自动删除最早的版本
- [ ] GET /api/admin/posts/{id}/history 正常返回
- [ ] 可以对比任意两个版本的差异
- [ ] 差异结果显示正确（新增/删除）
- [ ] 可以还原到指定版本
- [ ] 还原后创建新版本而非覆盖历史
