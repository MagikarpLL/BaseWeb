# M1-01 - 后端项目初始化

> 对应任务: [01-迭代计划.md](../../plan/01-迭代计划.md#m1-01-后端项目初始化)
> 涉及 Spec: [01-技术架构.md](../../spec/01-技术架构.md), [06-01-后端项目结构.md](../../spec/06-项目规范/01-后端项目结构.md)

## 验证目标

后端项目可独立运行，包含基础的项目结构和必要的配置文件。

## 测试用例

### TC-01: 项目结构验证

**前置条件：**
- 无

**测试步骤：**
1. 检查 `backend/` 目录存在
2. 检查 `pom.xml` 文件存在
3. 检查 `src/main/java/` 目录结构
4. 检查 `src/main/resources/` 目录结构

**预期结果：**
- 目录结构符合 `06-01-后端项目结构.md` 规范
- `pom.xml` 包含必要的依赖（Spring Boot, MyBatis, PostgreSQL, JWT, etc.）

### TC-02: 编译验证

**前置条件：**
- JDK 21 已安装
- Maven 3.8+ 已安装

**测试步骤：**
1. 进入 `backend/` 目录
2. 执行 `mvn compile`

**预期结果：**
- 编译成功，无错误
- 生成 `target/classes/` 目录

### TC-03: Spotless 格式化验证

**前置条件：**
- TC-02 通过

**测试步骤：**
1. 在 `backend/` 目录执行 `mvn spotless:check`

**预期结果：**
- 代码格式符合 Google Java Format 规范
- 无格式错误报告

### TC-04: Checkstyle 验证

**前置条件：**
- TC-02 通过

**测试步骤：**
1. 在 `backend/` 目录执行 `mvn checkstyle:check`

**预期结果：**
- 无代码规范违规
- Checkstyle 报告通过

### TC-05: 启动验证

**前置条件：**
- Docker 已启动
- PostgreSQL 容器已运行

**测试步骤：**
1. 配置 `src/main/resources/application-dev.yml` 数据库连接
2. 执行 `mvn spring-boot:run`
3. 等待服务启动
4. 访问 `http://localhost:8080/actuator/health`

**预期结果：**
- 服务启动成功
- Health 端点返回 `{"status":"UP"}`

## 验证方法

| 方法 | 命令 | 通过标准 |
|------|------|----------|
| 编译检查 | `mvn compile` | Exit code 0 |
| 格式检查 | `mvn spotless:check` | Exit code 0 |
| 规范检查 | `mvn checkstyle:check` | Exit code 0 |
| 启动测试 | `mvn spring-boot:run` | Health endpoint 返回 UP |
