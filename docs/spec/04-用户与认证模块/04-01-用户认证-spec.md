# 04-01-用户认证

> 对应 Requirement: [04-01-用户认证.md](../requirements/04-用户与认证模块/04-01-用户认证.md)

## 4.1.1 设计原则

| 原则 | 说明 |
|------|------|
| 安全第一 | 密码加密存储，错误次数限制，账户锁定 |
| 无感知刷新 | Access Token 过期自动刷新，用户无感知 |
| 会话安全 | Refresh Token 仅存 HTTP-only Cookie |

---

## 4.1.2 数据模型

### 登录 API

**接口**: `POST /api/auth/login`

**请求**:
```json
{
  "username": "admin",
  "password": "admin123"
}
```

**成功响应**:
```json
{
  "code": 200,
  "data": {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "Bearer",
    "expires_in": 900,
    "user": { "id": 1, "username": "admin", "role": "ADMIN" }
  }
}
```

**失败响应**:
```json
{
  "code": 401,
  "message": "用户名或密码错误"
}
```

### Token 刷新 API

**接口**: `POST /api/auth/refresh`

**请求**: 自动携带 HTTP-only Cookie

**成功响应**:
```json
{
  "code": 200,
  "data": {
    "access_token": "新token",
    "expires_in": 900
  }
}
```

**失败响应**:
```json
{
  "code": 401,
  "message": "登录已过期，请重新登录"
}
```

### 登出 API

**接口**: `POST /api/auth/logout`

## 4.1.4 关键逻辑

### 登录流程

```
1. 用户输入用户名/密码
2. 点击登录 → 禁用按钮 + loading
3. POST /api/auth/login
4. 成功 → 存储 access_token 到内存，存储 user 到 Pinia
5. 跳转 /admin
6. 失败 → 显示错误提示，解除按钮禁用
```

### Token 刷新机制

```typescript
// Axios 拦截器
api.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      // Token 过期，尝试刷新
      if (error.response.data.code === 40102) {
        try {
          const res = await api.post('/api/auth/refresh')
          const { access_token } = res.data.data
          // 更新内存中的 token
          authStore.setAccessToken(access_token)
          // 重试原请求
          return api(error.config)
        } catch {
          // 刷新失败，跳转登录
          router.push('/login')
        }
      }
    }
    return Promise.reject(error)
  }
)
```

### 密码错误次数限制

```typescript
// 后端逻辑
if (loginAttempts >= 5 && lastAttemptTime > now - 5min) {
  // 账户锁定
  return { code: 40104, message: "账户已锁定，请 15 分钟后重试" }
}
```

**锁定规则**：
| 条件 | 结果 |
|------|------|
| 5 分钟内错误 5 次 | 锁定 15 分钟 |
| 锁定期间登录 | 返回 40104 |

### 会话管理

| 状态 | 存储位置 | 过期策略 |
|------|----------|----------|
| accessToken | 内存（Pinia） | 15 分钟，自动刷新 |
| user | 内存（Pinia） | 跟随 accessToken |
| refreshToken | HTTP-only Cookie | 7 天，自动刷新 |

## 4.1.6 验收标准

- [ ] 正确用户名密码可以登录成功
- [ ] 错误用户名密码返回"用户名或密码错误"提示
- [ ] 连续 5 次错误后账户锁定 15 分钟
- [ ] 登录成功后跳转 `/admin`
- [ ] Access Token 过期后自动刷新（用户无感知）
- [ ] 刷新失败后跳转登录页并显示提示
- [ ] 访问 `/admin` 未登录时重定向 `/login`
- [ ] 已登录访问 `/login` 重定向 `/admin`
- [ ] 登出后清除所有状态并跳转 `/login`
- [ ] 刷新页面后登录状态保持（通过 Refresh Token）
